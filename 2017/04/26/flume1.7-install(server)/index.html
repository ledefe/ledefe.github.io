Flume 1.7 安裝（服务聚合端）

该版本是基于flume 1.7版本，扩展兼容elasticsearch5.3.0

资源准备

资源： flume.zip

部署步骤

指定安装目录

mkdir -p /opt/flume_server

接下来的操作都在/opt/flume_server下操作

zip包准备

  • 该flume.zip已经支持flume es5.3.0 sink
  • 上传至/opt/flume_server目录下

cd /opt/flume_server
rz -y

解压zip包

  • 在/opt/flume_server目录下解压

unzip flume.zip

定义配置文件

  • 配置文件都放置到conf中去

mkdir -p cloud/conf
cd cloud/conf

  • 这里分别定义三个文件：4145.conf 4144.conf log4j2.xml
  • 4145.conf 4144.conf 分别对应两个agent服务。
  • 对外提供avro服务，并最终持久化到es服务中。
  • 目前文件已定义好，只需更改对应es服务相关信息和avro提供的端口
  1. 4145.conf文件
  1. 4144.conf文件
  1. log4j2.xml文件

${sys:flumeInstance} 主要是从系统变量里面获取值，动态设置日志输入目录，通过 -DflumeInstance=xxx进行设置，已在flume_cloud_start.sh中定义好

##修改脚本配置

  1. 配置flume_cloud_start.sh

  • 设置实例路径

    找到 instance_root_dir变量设置实例目录
    默认设置为 /opt/flume_server/instance

  • 设置运行配置目录

    设置cloud目录，当前cloud是放在和该执行脚本同级上，可根据需求自行修改

    nohup $dir/bin/flume-ng agent -c $dir/cloud/conf -f $dir/cloud/conf/${con_file}.conf -DflumeInstance=$flumeInstance -n a1 >${flumeInstance}/opt.log 2>&1 &

  • 授权

  1. flume_cloud_stop.sh脚本

  • 设置实例路径

    找到 log_dir变量设置实例目录
    可根据需要更改，默认是在/opt/flume_server/instance下），必须和flume_cloud_start.sh里设置的目录一样
    log_dir=/opt/flume_server/instance

  • 授权

运行

添加至开机自启动

追加以下内容

错误解决

nohup: failed to run command ‘/opt/flume_server/flume/bin/flume-ng’: Permission denied

flume-ng未授权

开机启动 找不到javahome

可直接在flume_cloud_start.sh中加入JAVA_HOME
