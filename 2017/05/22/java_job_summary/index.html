java工作技术总结

redis的使用

redis是一款nosql数据库，一般用来做临时缓存。也可以用来做消费者和提供者的模型。
改造过一款基于redis的后台任务组件，jesque能把任何类或模块作为任务在后台执行，且自带前台管理功能，方便查看执行情况。用于程序间解藕。

monogodb的使用

也是一款nosql数据库，自由的文档模式，面向集合存储，适合存储文档和json形式的数据。它提供丰富的查询语法。
能够集群和故障自动转移，用过作为图文存储的介质。
缺点：锁粒度比较粗，2.2以前的版本，只有全局锁，2.2之后，提升为集合锁。例如如果一个 mongod 实例上有 5 个库，如果只对一个库中的一个集合执行写操作，那
么在写操作过程中，这个库被锁；而其它 4个库不影响。相比 RDBMS 来说，这个粒度已经算很大了！
