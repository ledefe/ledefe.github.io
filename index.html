nginx安装笔记

转载
2017-10-11
ledefe
linux
nginx

环境

centos 7

依赖安装

yum -y install pcre-devel
yum -y install openssl openssl-devel

安装

cd /tmp
wget http://nginx.org/download/nginx-1.13.6.tar.gz
tar -zxvf nginx-1.13.6.tar.gz
cd nginx-1.13.6
./configure –prefix=/usr/local/nginx

命令

启动：/usr/local/nginx/sbin/nginx
停止/重新加载：/usr/local/nginx/sbin/nginx -s stop(quit|reload)
验证配置文件是否合法：/usr/local/nginx/sbin/nginx -t
命令帮助：/usr/local/nginx/sbin/nginx -h

简介

默认制作的脚本安装至/usr/local/logstash_server/logstash-5.3.0下,版本为：5.3.0

默认的java安装目录为：/usr/local/java

制作步骤

  • 1.准备logstash-5.3.0.tar.gz原装包
  • 2.解压至logstash_server下
  • 3.复制脚本tail_start.sh|tail_stop.sh至logstash_server/logstash-5.3.0下
  • 4.解压复制附件中的config目录至对应logstash_server/logstash-5.3.0/config目录
  • 5.压缩 tar -czvf logstash_server.tar.gz logstash_server
  • 6.放置logstash_server.tar.gz和logstash_install.sh同级至任何目录执行
  • 7.安装命令：sh logstash_install.sh -h 127.0.0.1:9200
    -h 为es的http 地址

elasticsearch 优化篇之-调整索引速度（来自官网）

原创
2017-09-08
ledefe
elasticsearch
elasticsearch

使用批量请求

批量请求将产生比单一文档索引请求更好的性能。为了了解批量请求的最佳大小，您应该使用单个分片在单个节点上运行基准测试。
首先尝试一次索引100个文档，然后在每个基准运行中索引100个文档，然后200个等于大量请求中的文档数量。
当索引速度开始升高时，您就知道您已达到数据批量请求的最佳大小。请注意，太大的批量请求可能会使集群在许多并发发送时处于内存压力之下，
因此建议避免每个请求超过几十MB，即使较大的请求似乎表现更好。

EFFK-日志采集方案II

原创
2017-08-18
design
EFK 日志采集

简介

EFLK是Elasticsearch、filebeat、logstash、kibana的简称。

Elasticsearch:是个开源实时分布式搜索引擎。它的特点有：分布式，零配置，自动发现，索引自动分片，索引副本机制，restful风格接口，多数据源，自动搜索负载等。

filebeat: elastic官方维护的日志采集程序，基于go语言编写，资源消耗小。

logstash：作为日志的汇总中心，其具有丰富的转换插件，提供日志转换。

kibana:是一个为ElasticSearch 提供的日志分析的 Web 接口。可使用它对日志进行高效的搜索、可视化、分析等各种操作。
