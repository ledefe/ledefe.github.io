Flume 1.7 安裝（服务聚合端）

该版本是基于flume 1.7版本，扩展兼容elasticsearch5.3.0

资源准备

资源： flume.zip

部署步骤

指定安装目录

mkdir -p /opt/flume_server

接下来的操作都在/opt/flume_server下操作

zip包准备

  • 该flume.zip已经支持flume es5.3.0 sink
  • 上传至/opt/flume_server目录下

cd /opt/flume_server
rz -y

解压zip包

  • 在/opt/flume_server目录下解压

unzip flume.zip

定义配置文件

  • 配置文件都放置到conf中去

mkdir -p cloud/conf
cd cloud/conf

  • 这里分别定义三个文件：4145.conf 4144.conf log4j2.xml
  • 4145.conf 4144.conf 分别对应两个agent服务。
  • 对外提供avro服务，并最终持久化到es服务中。
  • 目前文件已定义好，只需更改对应es服务相关信息和avro提供的端口
  1. 4145.conf文件
  1. 4144.conf文件
  1. log4j2.xml文件

${sys:flumeInstance} 主要是从系统变量里面获取值，动态设置日志输入目录，通过 -DflumeInstance=xxx进行设置，已在flume_cloud_start.sh中定义好

##修改脚本配置

  1. 配置flume_cloud_start.sh

  • 设置实例路径

    找到 instance_root_dir变量设置实例目录
    默认设置为 /opt/flume_server/instance

    设置cloud目录，当前cloud是放在和该执行脚本同级上，可根据需求自行修改

    nohup $dir/bin/flume-ng agent -c $dir/cloud/conf -f $dir/cloud/conf/${con_file}.conf -DflumeInstance=$flumeInstance -n a1 >${flumeInstance}/opt.log 2>&1 &

    找到 log_dir变量设置实例目录
    可根据需要更改，默认是在/opt/flume_server/instance下），必须和flume_cloud_start.sh里设置的目录一样
    log_dir=/opt/flume_server/instance

Flume 1.7.0 supporting ElasticSearch 5.3.0

原创
2017-04-11
elk
Flume ElasticSearch

环境说明

实现git地址:https://github.com/ledefe/flume-elasticsearch-sink

Flume1.7下载地址：http://flume.apache.org/download.html

步骤说明

1.下载该git源码初始化环境

导入至eclipse中去，执行install maven命令生成最终的jar包

mvn install

target目录下获取 flume-elasticsearch-sink-0.0.1-SNAPSHOT.jar 拷贝至flume的lib目录

2.获取依赖的包

执行以下命令，会在target目录下生成dependencies目录，将得到的jar拷贝至flume的lib目录中

mvn dependency:copy-dependencies

3.去除flume的lib目录下的冲突的包:

  • jackson-annotations-2.3.0.jar

  • jackson-core-2.3.1.jar

  • jackson-databind-2.3.1.jar

  • jopt-simple-3.2.jar

  • log4j-1.2.17.jar

  • netty-3.9.4.Final.jar

  • guava-11.0.2.jar

  • flume-ng-elasticsearch-sink-1.6.7.jar

  • slf4j-log4j12-1.6.1.jar

4.更新日志文件 log4j to log4j2

  • 将git中的 log4j2.xml 拷贝至 flume/conf下

搜索设计

背景

为5000个ktv商家提供曲库搜索服务，每个商家差不多有15k的曲库信息

需求

  • 与商家绑定，查询商家的歌曲信息

设计思路

  • 每个商家的曲库信息不一，通过变更的曲库信息文件通过ftp上报至ftp服务
  • 通过清洗插件对每个商家的曲库信息做比对，对有变更的曲库信息做更新操作
  • 采用es或solr搜索引擎存储

扩展性
